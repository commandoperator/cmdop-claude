# v0.1.60 — Robust SKILL.md allowed-tools parsing

**Date:** 2026-03-10

## Bug Fixes

- `SkillFrontmatter` — `allowed-tools` field now uses `Annotated[list[str], BeforeValidator(_coerce_tools)]`
- Handles all real-world formats: comma-separated string, YAML list, bracket list, `Bash(git:*)` patterns, `None`
- `extra="ignore"` on `SkillFrontmatter` — unknown frontmatter keys from third-party skills no longer crash the parser
- Added `user-invocable`, `argument-hint`, `context`, `agent` fields per Claude Code spec

## Improvements

- `metadata: dict[str, object]` (was `dict[str, str]`) — values can now be nested objects
