# v0.1.72 — Non-blocking background scan

**Date:** 2026-03-10

## Bug Fixes
- `_maybe_auto_scan` in `hook.py` — scan now runs as detached `subprocess.Popen` instead of blocking synchronously; eliminates `UserPromptSubmit hook error` caused by LLM timeouts and context overflows during `inject-tasks`
- `scan` CLI command — now converts review items to tasks after generating review (previously only happened in the blocking path)
- Hook returns in ~1.4s regardless of LLM response time; scan result appears in next prompt via `get_pending_summary`
