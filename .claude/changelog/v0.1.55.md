# v0.1.55 — Raise content budget limits

**Date:** 2026-03-10

## Improvements

- `scan_doc_files()` — removed arbitrary 60-file cap; all files scanned with priority order preserved
- `_build_contents_block()` — `MAX_PER_FILE_CHARS` raised to 8k, `MAX_TOTAL_CHARS` raised to 400k (~100k tokens)
- Files exceeding budget remain in metadata list so LLM knows they exist
- Skip threshold raised to 500kb (pathologically large files only)
