# v0.1.65 — Auto-update system

**Date:** 2026-03-10

## Features
- `UpdateService` — checks PyPI JSON API for newer version, runs `pip install --upgrade` as non-blocking background process via `subprocess.Popen`
- `_maybe_auto_update()` in `inject-tasks` hook — fires at most once per 6 hours, skips if `.sidecar/.lock` exists (review/fix in progress), prints one-line notice to Claude context when upgrade is launched

## Improvements
- `hook.py` — `_handle_inject_tasks` now calls `_maybe_auto_update` before `_print_version_line`
- README — added Auto-update section and `services/updater/` to architecture

## Tests
- 13 new tests in `tests/services/test_update_service.py` covering `is_newer`, `get_installed_version`, `fetch_latest_version` (mock network), and `launch_upgrade` (mock Popen)
